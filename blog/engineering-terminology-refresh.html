Engineering Terminology Refresh: Systems, Performance, Data & Representation

Today, I'm revisiting fundamental concepts in software and systems engineering. Whether it's understanding the intricacies of deterministic versus non-deterministic systems or mastering performance optimizations like batching and vectorization, these principles are crucial for building robust and scalable AI-driven applications.

Refreshing on Engineering Terminology

System Behavior & Correctness

  • guarantees
  • reliability
  • predictability
  • correctness properties

Terms:

Deterministic: Same Input -> Same Output

Traditional Software mostly deterministic

Non-Deterministic

random.random()
datetime.now()
thread scheduling
LLM sampling

LLMs are often probabilistic / non-deterministic due to:
- sampling
- temperature
- model variance
- GPU execution variance

AI systems are inherently probabilistic.
AI engineered systems work to combine Deterministic rules + Probabilistic AI inference.

if confidence > 0.95:
    auto-approve
else:
    human review

Probabilistic - Outputs are likelihood-based, not guaranteed.

Stochastic - randomness intentionally involved.

examples)
- SGD
- Sampling
- Exploration Policies

Invariant - A condition that must always remain true.

Database Invariant - user_id must exist.
Financial Invariant - balance cannot go negative.
Queue Invariant - message processed once.
AI Systems
Safety Invariant - human approval required before payout.
Workflow Invariant - policy must have valid carrier.
Agent Invariant - planner cannot directly execute shell commands.

Consistency

  • database consistency
  • logical consistency
  • eventual consistency
  • output consistency

Idempotent - Running something multiple times yields same result.

PUT /user/5/status=active

Referential Transparency - same inputs -> same outputs without side effects.

Ephemeral - short lived.

examples)
- containers
- tokens
- workers

Durable - persistent/recoverable.

Mutable vs Immutable - can vs can't change

Critical For:
- concurrency
- functional programming
- event sourcing

Performance & Scalability

Amortized - average cost over many operations.

Dynamic Array append.

Most appends: O(1)
Occasionally: resize array -> O(n)
Amortized (averaged over many appends) - amortized O(1)

Throughput vs Latency

Throughput - How much work per unit time.
Latency - How long one request takes.

Backpressure - system slowing intake because downstream overloaded.

Important For:
- queues
- streaming
- inference systems

Batching - grouping multiple operations together to reduce overhead and improve throughput.

  • Bulk database inserts
  • Batch inference requests
  • Kafka consumer batches
  • Writing many rows in one transaction

Tradeoff:
- Higher throughput
- Increased latency for individual items

AI/ML:
- GPU inference is often heavily batch optimized.
- Training commonly processes batches of tensors simultaneously

Parallelism - multiple operations executing at the same time

Examples)
- Multi-core CPU processing
- GPU tensor operations
- Parallel API workers
- MapReduce jobs

Concurrency - multiple tasks making progress during overlapping periods of time.

Examples)
- async web servers
- thread scheduling
- event loops
- cooperative multitasking

Concurrency often focuses on:
- coordination
- shared resources
- synchronization
- scheduling

Vectorization - applying operations across arrays/tensors simultaneously instead of iterating item-by-item.

Examples)
- NumPy operations
- GPU tensor math
- SIMD instructions
- Matrix multiplication

# instead of:
for x in arr:
    x += 1

# vectorized
arr += 1

Important in ML training, inference, scientific computing, GPU acceleration

Quantization - reducing model precision to reduce memory and improve inference speed.

examples)

  • FP32 -> FP16
  • FP16 -> FP8
  • 8-bit -> 4-bit quantization

Tradeoff:
- smaller/faster models
- potential accuracy loss

Important for:
- Ollama
- local inference
- edge deployment
- GPU memory optimization

Fan-out - distributing work from one source to many downstream workers/services.

Examples)
- Queue distributing jobs to workers
- Broadcast messaging
- Parallel inference requests
- Searcg across multiple indexes

Often used to:
- scale horizontally
- parallelize workloads

Fan-in - combining results from many workers/services into a single result.

Examples)
- Aggregating worker outputs
- Combining map results
- Multi-agent synthesis
- Search result aggregation

Commonly paired with:
- fan-out -> processing -> fan-in

Sharding - splitting data across multiple machines/databases based on some shard key

Examples)
- user_id % N
- geographic region
- tenant-based partitioning

Benefits:
- horizontal scaling
- reduced load per database

Tradeoffs:
- cross-shard joins become difficult
- rebalancing complexity
- operational overhead

Partitioning - dividing data into smaller logical sections.

Can be:
- horizontal partitioning (rows)
- vertical partitioning (columns)
- time partitioning
- range partitioning

Examples)
- monthly telemetry tables
- partition by customer_id
- partition by date

Benefits:
- faster queries
- easier maintenance
- archive management

Data & Representation

Discrete - made of separate/countable states instead of continuous values.

Examples)
- integers
- tokens
- graph npdes
- vocabulary IDs
- events

opposite of continuous

Continuous - values exist on a smooth spectrum rather than separate countable states.

Examples)
- floating point tensors
- audio waveforms
- probability distributions
- embeddings

Continuous systems often use:
- calculus
- gradients
- optimization

Latent - hidden/internal representation learned by a model.

Examples)
- latent variables
- latent embeddings
- latent feature space

LLms and neural networks learn latent representations internally.

Latent spaces often encode:
- meaning
- similarity
- structure

Embedding - continuous vector representation of data or meaning.

Exmaples)
- text embeddings
- image embeddings
- sentence embeddings

Embeddings allow:
- semantic search
- clustering
- similarity comparisons
- retrieval systems

Very important in:
- RAG
- vector databases
- recommendation systems

Vector Space - mathematical space where vectprs exost amd distances/relationships can be measured.

Embeddings exist within vector spaces.

Important properties:
- distance
- similarity
- direction
- clustering

Examples)
- cosine similarity
- nearest neighbor search
- semantic clustering

Serialization - converting structured data into a transportable/storable format.

Examples)
- JSON
- protobuf
- MessagePack
- Avro

Used for:
- APIs
- Queues
- Persistence
- Networking

Opposite of de-serialization

Canonical - official/source of truth representation.

Examples)
- canonical database record
- canonical API schema
- canonical event format

Important for:
- consistency
- synchronization
- deduplication

Semantic - meaning-aware rather than syntax-aware.

examples)
- semantic search
- semantic similarity
- semantic clustering

Sematic systems attempt to understand:
- intent
- meaning
- relationships

Instead of exact keyword matching.

Tokenization - breaking data into smaller units called tokens.

Examples)
- words
- subwords
- characters
- byte-pair encodings

LLMs tokenize text before inference.

Example)

"ChatGPT is useful"

# might become
["Chat", "G", "PT", "is", " useful"]

Different tokenizers produce different token boundaries.

Feature - measurable property/input used by a model

Examples)
- age
- transaction count
- embedding dimensions
- word frequency
- pixel intensity

Features are inputs into:
- classifiers
- regressions
- neural networks

Dimensionality = number of values/features used to represent data.

Examples)
- 3D coordinates
- 768-dimensional embeddings
- feature vectors

Higher dimensionality:
- can encode richer information
- increases computational complexity

High-dimensional spaces often behave unintuitively:
- sparse data
- curse of dimensionality
- difficult nearest-neighbor search

Heuristic- Rule-of-thumb solution.

Not guaranteed optimal.

Often used when:
- exact optimizations is too expensive
- search space is too large
- approximate solutions are acceptable

Examples)
- retrieve top 5 search results
- greedy algorithm
- recommendation ranking
- beam search

Tradeoff:
- Faster/simpler
- Less optimal/correct

Latent - hidden/internal representation.

examples)
- latent space
- latent features
- latent variables

Neural networks learn latent representations internally.

Laten spaces often encode:
- meaning
- relationships
- similarity
- structure

Embeddings are latent representations.

Emergent Behavior - Complex behavior arising unexpectedly from simpler systems.

Examples)
- Agent planning behavior
- LLM reasoning chains
- Multi-Agent collaboration
- self-correction loops

Emergent behavior is often:
- difficult to predict
- not explicitly programmed
- prodcued from scale/interactions

Important in:
- agents
- distributed systems
- neural networks

Convergence - System stabilizes towards a solution or steady state.

ML:
- training converges
- loss stabilizes

Distributes Systems:
- Eventual convergence
- Replicas synchronize over time

Optimization:
- algorithm approaches local/global optimum

Failure modes:
- divergence
- oscillation
- instability

Tensors - multi-dimensional numerical arrays used heavily in ML computation.

Examples)
- scalars (0D tensor)
- vectors (1D tensor)
- matrices (2D tensor)
- higher-dimensional tensors

Deep learning frameworks:
- PyTorch
- TensorFlow
- JAX

Tensors are optimized for:
- GPU computation
- parallel math
- vectorized operations

Classifiers - models that assign categories/labels to inputs.

Examples)
- spam detection
- fraud classificatioj
- image recognition
- sentiment analysis

Classifier outputs are often:
- probabilities
- confidence score
- labels

Examples)

email -> spam
transaction -> fraud
image -> cat

Common classifier types:
- logistic regression
- random forest
- neural network
- transofrmer classifier

Hallucination - model generates plausible but incorrect information.

Hallucinations are often:
- statistically plausible
- high-confidence
- factually incorrect

Causes:
- insufficient grounding
- weak retrieval
- probabilistic generation
- training data limitations

Mitigations:
- RAG
- grounding
- evals
- tool calling
- structured outputs

Grounding - constraining model outputs using external truth/data/context.

Examples)
- retrieval augmented generation (RAG)
- database lookups
- web search
- tool calling

Grounding reduces:
- hallucinations
- unsupported claims
- fabricated outputs

Important in:
- enterprise AI
- agents
- production AI systems

Evals (Evaluation Engineering) - testing againt benchmark datasets.

Testing:
- prompts
- workflows
- agents
- outputs

Examples)
- golden datasets
- regression testing
- benchmark scoring
- human review pipelines

Topology - Structure/layout of a system/network

Examples)
- microservice topology
- network topology
- cluster topology
- graph topology

Topology describes:
- connectivity
- relationships
- communication paths

Important in:
- distributed systems
- Kubernetes
- agent orchestration
- networking

Fan-out / Fan-in - Parallel distribution and recombination patterns.

Fan-out:
- distribute work to many workers/services

Fan-out:
- combine many results into one result

Examples)
- distributed search
- parallel inference
- map/reduce systems
- multi-agent workflows

Pattern:

request
   ↓
fan-out
   ↓
parallel workers
   ↓
fan-in
   ↓
aggregated result

Meta-Prompting - prompts that generate, improve, evaluate, or orchestrate other prompts.

Examples)
- prompt optimization
- self-reflection prompts
- prompt generation systems
- DSPy workflows
- recursive prompting

Common patterns:
- critique/revise loops
- planner/executor prompts
- chain-of-thought refinement
- automatic prompt optimization

Meta-prompting is heavily used in:
- agents
- orchestration systems
- evaluation systems
- autonomous workflows
