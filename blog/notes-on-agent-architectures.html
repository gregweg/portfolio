Notes on Agent Architectures

Today, I'm diving into the intricacies of building reliable and cost-effective AI agents that can perform a variety of tasks, from candidate screening to customer support. The core challenge is integrating multiple components like planning, tool use, and memory management while mitigating common failure modes.

Agent Failure Modes

Hallucination - agents like to invent facts

Migitation:

RAG
citations
tool verification
structured outputs

Infinite Loops - agents repeatedly call tools.

Mitigation:

max iterations
budgets

Tool Abuse - agents repeatedly invokes expensive APIs.

Mitigation:

tool limits
rate limits
cost controls
approval gates

Context Window Problems - agents forget earlier information.

Mitigation:

summarization
memory stores
retrieval

Prompt Injection - agent loses earlier information.

Mitigation:

tool permissions
sanboxing
input filtering
human review

Workflow Orchestration

Orchestration Provides

  • state persistence
  • retries
  • recovery
  • timeouts
  • scheduling
  • human approval
  • branching logic

Temporal - write workflows as code.

Temporal handles:

state
retries
durability
recovery

ex)

@workflow
def hiring_workflow():
    candidate = search_candidates()

    approved = wait_for_manager()

    if approved:
        schedule_interview()

If worker crashes, workflow resumes instead of restarting.

Without Temporal:

DB tables
cron jobs
retry queues
status flags
state machines

Temporal centralizes this.

Key Temporal Concepts

Workflow - Long-lived durable process.
Activity - Individual unit of work.

search_candidates()

Retry Policy - Automatic retries.
Signals - External events. (Ex. actions like, manager approved)
Queries - Read workflow state.
